# 模拟登录系统

这是一个为 v0 环境设计的模拟登录系统，无需真实的身份验证即可运行应用。

## 功能特点

- 🚀 **无需真实认证**：适用于演示和开发环境
- 👥 **预设用户**：提供3个预设的模拟用户
- 📝 **自定义登录**：支持创建自定义用户
- 💾 **本地存储**：用户状态保存在浏览器本地存储中
- 🎨 **完整UI**：包含登录界面和应用主界面

## 使用方法

### 1. 访问模拟应用
```
http://localhost:3000/mock
```

### 2. 快速登录
选择以下预设用户之一：
- **张妈妈** (parent1@example.com) - 孩子：小明，6岁
- **李爸爸** (parent2@example.com) - 孩子：小红，8岁  
- **王妈妈** (parent3@example.com) - 孩子：小刚，5岁

### 3. 自定义登录
- 使用任意邮箱和密码登录
- 也可以点击 GitHub 或 Google 按钮进行模拟社交登录

### 4. 应用功能
登录后可以体验：
- 📊 仪表盘：查看孩子的习惯完成情况
- 🎯 习惯管理：管理孩子的日常习惯
- 🎁 奖励商店：积分兑换系统  
- 📈 成长进度：查看成长数据

## 文件结构

```
components/
├── auth/

├── mock-app.tsx                 # 主应用组件
└── mock-sidebar.tsx             # 侧边栏组件

app/
└── mock/
    └── page.tsx                 # 模拟应用入口页面
```

## 组件说明

### MockAuthProvider
- 管理用户认证状态
- 提供 signIn/signOut 方法
- 使用 localStorage 持久化用户状态

### MockLogin  
- 提供快速登录选项
- 支持表单登录和社交登录模拟
- 美观的 UI 设计

### MockApp
- 主应用容器
- 根据认证状态显示登录页面或应用内容
- 包含侧边栏导航

## 技术栈

- **React 18**：前端框架
- **TypeScript**：类型安全
- **Tailwind CSS**：样式框架
- **Shadcn/ui**：UI 组件库
- **Lucide React**：图标库

## 开发说明

这个模拟系统完全独立于真实的认证系统，可以在以下场景使用：

1. **v0 环境演示**：无需配置真实的 OAuth 或数据库
2. **开发测试**：快速切换不同用户进行功能测试  
3. **产品演示**：向客户展示应用功能
4. **原型开发**：快速验证应用流程

## 注意事项

- 所有数据都是模拟的，不会保存到真实数据库
- 用户状态仅保存在浏览器本地存储中
- 刷新页面会保持登录状态，除非手动退出登录
- 适用于演示环境，不建议在生产环境使用

## 定制化

可以通过修改以下文件来定制：

- `MOCK_USERS` 数组：添加更多预设用户
- `sidebarItems` 数组：修改导航菜单
- `renderPageContent` 方法：添加更多页面内容
- CSS 样式：调整界面外观

## 与真实应用的集成

要在真实应用中集成，只需：

1. 添加环境变量控制是否使用模拟登录
2. 在 app/layout.tsx 中条件性地使用 MockAuthProvider
3. 在路由中添加模拟模式的判断逻辑
